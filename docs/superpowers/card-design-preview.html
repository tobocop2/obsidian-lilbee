lilbee model card — design directions

Three takes on the catalog card. Each one is a self-consistent system — pick one and we'll port it into the plugin. The grid below mirrors the real catalog layout (~260px columns) so spacing and overflow behave the way they will in production.

A · Editorial spec left rail signals fit · clear hierarchy · pills below name

The card itself is the indicator. A thin colored rail on the left says fits/tight/won't-run; the rest of the card stays calm. Specs in monospace feel like data, not chrome. Best for grids of 2–3 columns where each card needs to read like a record.
